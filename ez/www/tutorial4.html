Tutorial 4. Handling Post with Jquery

Result


If we click the button, it will show alert message from server.

index.html

<html>
	<head>
		<script src="jquery.js"></script>
		<script>
			function on_click() {
				$.post('result_view.html', {name : 'abc', num : 123}, 
					function(data, status) { alert(data); }
				);
			}
		</script>
	</head>
	<body>
	<button onclick='on_click()'>post with jquery</button><br>
	</body>
</html>

Yes, we don't have jquery_post file in our web directory. So content_ is empty this time. If there is any file in our html directory, the eZ Framework library will fill the content_ variable with it. If it cannot find one, it will leave content_ empty. And, requested document_ can be anything. Not like jsp, asp, it can be html file or sometimes non-existing file. We can use this just as a variable. In this case, to verify a jquery request.

우리는 jquery_post란 템플릿 파일이 우리의 템플릿 디렉토리에 없다. 그러므로, content_ 는 텅 비어 있을 것이다. requested_document_는 어떤 것이든 될 수 있다. 다른 서버측 프레임워크들에서는 jsp 혹은 asp로 서버측 처리를 위한 페이지는 따로 정해져 있었지만, eZ 프레임워크는 그렇지 않다. eZ 프레임워크는 평범한 html 파일이나 존재하지 않는 문서도 요청될 수 있다. content_의 내용이 어떤 내용으로 채워질 것인지, 비어 있을 것인지만 정할 뿐이다.
