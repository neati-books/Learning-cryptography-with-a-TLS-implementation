Reference

eZ 프레임워크 최대의 강점은 채 한 페이지도 되지 않는 레퍼런스입니다.

The strongest point of eZ Framework is that the whole reference is less than a page

서버 사이드 웹 프로그래밍을 하기 위해서, 개발자는 단지 단 두개의 클래스와 3개의 변수, 7개의 함수만 알면 됩니다.

eZ Framework is made of just two classes and 3 variables and 7 functions. Developer need to know just these to make a web site.

WebSite 클래스 함수

WebSite Class Functions

WebSite 클래스는 단 세 개의 편의 함수를 제공합니다. 이 함수는 content_값을 변화시키는 데에 사용합니다.

WebSite class offer 3 utility functions. swap, append, prepend. These functions are used to change the content_ variable.

public :
static void init(string dir) : 디렉토리 dir에서 템플릿 파일들을 로드한다.
static void add_header(string document, string header) : document의 HTML response에 헤더를 붙인다. CORS등에 사용.
protected :
bool swap(string a, string b) : content_에서 a를 찾아서 b로 바꿉니다. 바꾸었으면 true 리턴
bool append(string a, string b) : content_에서 a를 찾아서 그 바로 뒤에 b를 붙인다.
bool prepend(string a, string b) : content_에서 a를 찾아서 그 바로 앞에 b를 붙인다.
public :
static void init(string dir) : Load template files from directory dir
static void add_header(string document, string header) : attach header to HTML response
protected :
bool swap(string a, string b) : find a in content_ and change it to b, return true if swap succeded.
bool append(string a, string b) : find a in content_ and append b after a
bool prepend(string a, string b) : find a in content_ and prepend b before a

WebSite 클래스 멤버 변수

Member variables of WebSite class

eZ 프레임워크는 사용하기 위해 외워야 할 것이 거의 없습니다. MySql 데이터베이스를 위한 SqlQuery 클래스, 원격 접속을 담당하는 Server 클래스, 웹사이트의 서버측 로직을 담당하는 WebSite 클래스가 다입니다. Server 클래스는 생성자에서 포트번호를 정해주고, start함수로 WebSite 클래스 혹은 WebSite 클래스에서 파생된 자식 클래스를 실행하면 됩니다. WebSite 클래스는 내부적으로 사용하며 프로그래머가 알아야 하는 세개의 멤버 변수가 있습니다.

eZ Frammework does not have much to memorize to develop. SqlQeury class for MySql input and output, Server class for networking, WebSite class for server side logic. These are all you should know. Server class specify the port number in constructor, and run the WebSite class or a class inherited from WebSite class with start function. There are three member variables you should know in WebSite class.

protected :
string requested_document_ : 요청된 페이지 이름
string content_ : 요청된 페이지의 템플릿 값에 따라 초기화된 문자열
map<string, string> nameNvalue_ : post, get, 혹은 multifile-form data 에서 온 파라미터와 그 값들
protected :
string requested_document_ : requested document by the browser
string content_ : initialized with template of requested document
map<string, string> nameNvalue_ : parameters and the values of post, get or multiform data
