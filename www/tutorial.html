eZ 프레임워크는 라이브러리는 사용되어야 할 것이지, 배워야 할 것이 아니다라는 모토를 기반으로 만들어졌습니다. eZ 프레임워크는 매우 쉬운 서버측 웹개발 프레임워크입니다. eZ 프레임워크로 개발하기 위해 외워야 할 것은 몇 가지 밖에 안됩니다. 나머지는 프로그래머의 응용력에 달려 있습니다. eZ 프레임워크는 단순한 웹 개발 환경을 제공합니다. 프로그래머는 한 명의 접속자와 연속적인 메세지 교환을 한다고 생각하고 웹 앱을 개발하면 됩니다. 웹사이트를 하나의 클래스로 정의하기에 구조적인 프로그래밍을 하기에도 용이합니다. C++은 성숙하고 다양한 기능을 제공하며, 빠른 언어임에도 불구하고, 웹 개발에 있어서는 거의 사용되지 않았습니다. 그 가장 큰 이유는 웹 개발은 일반적인 소프트웨어 개발과는 다르게, 웹에 접근하는 사용자가 보는 페이지 별로 분절되어 있기 때문입니다. 그렇기 때문에 본격적인 언어보다는 스크립트형 언어가 웹 개발에 더욱 많이 쓰이고 있습니다. 또, 웹 개발에서는 로그인 등의 상태 정보를 저장하기 위해서 쿠키를 사용하는 등의 복잡한 절차를 이용해야 합니다. eZ 프레임워크는 하나의 브라우저가 연속적으로 연결되어 있는 것과 같은 Stateful한 가상적인 환경을 제공합니다. 이 가상적인 환경이 페이지간의 분절을 없애줍니다. 이 덕분에 프로그래머는 편하게 C++의 변수에 여러 페이지에 걸쳐 기억해야 할 정보를 할당할 수 있습니다. 더 이상 로그인 되어 있는 상태인지 확인하기 위해 쿼리를 날린다거나, 쿠키를 확인하는 복잡한 작업을 할 필요가 없습니다. 그저 멤버 변수나 지역 변수를 확인하면 됩니다. eZ 프레임워크는 HTML, CSS, Java Script 등을 템플릿으로 사용할 수 있으며, 복잡한 템플릿 문법이 없습니다. 단지 swap, append 등의 함수를 제공합니다. 정규표현식에 친숙하다면 이를 사용해서 문자열을 조작할 수도 있습니다. eZ 프레임워크는 자신의 웹서버로 돌릴 실행파일을 직접 만듭니다. 아파치나 톰캣 같은 덩치 큰 서버 프로그램을 돌릴 필요가 없습니다. eZ 프레임워크는 https 암호화와 가상 환경을 제공하는 미들 서버와 웹 서버로 나뉘어 있습니다. 미들서버는 프레임워크에서 곧장 사용할 수 있는 기성품의 형태로 제공합니다. 개발자는 웹 서버만을 개발하면 됩니다. 이 미들서버를 사용하면 복잡하게 설정을 할 필요가 없이 쉽게 암호화 된 웹사이트를 구현할 수 있습니다. 이 미들서버는 만약 아파치 서버와 외부 브라우저 사이에 위치시킨다면 아파치 서버의 설정에 아무런 변경 없이 암호화를 제공할 수 있을 정도로 유연합니다. 물론, eZ 프레임워크로 개발한 웹 서버와 브라우저 사이에 위치시킨다면, Stateful한 편리한 개발 환경까지 제공합니다. eZ 프레임워크는 MySql 데이터 베이스에 데이터를 입력하고 출력하는 편의 클래스를 제공하고 있습니다. 그 외의 데이터 베이스의 경우에는 개발자가 스스로 라이브러리를 사용하여 링크하면 됩니다. python이나 다른 언어를 사용하여 접근할 수도 있습니다. 웹 서버 프로그램이 다른 실행 파일이나 스크립트 파일을 실행시키거나, pybind 등을 이용하여 파이선 함수를 부르는 것은 그다지 어렵지 않습니다. eZ 프레임워크는 "웹서버는 문자열을 받아서 처리한 후, 문자열을 돌려주는 것이다"라는 매우 단순한 통찰을 기반으로 만들어졌습니다. eZ 프레임워크는 문자열을 받아서 그 요청이 GET이던 POST이던 multi-form file 업로드이던 ajax 이건 상관없이 변수이름과 그 값을 nameNvalue_라는 일종의 map에 저장합니다. 요청된 페이지는 requested_document_라는 변수에 저장합니다. 그리고, 요청된 페이지의 템플릿이 있을 경우에는 그 내용을 content_에 저장합니다. 그리고, 이 content_값은 브라우저에게 보내질 것입니다. 그러나, 브라우저에게 이 값을 보내기 전에 서버단에서의 처리를 위하여, process함수를 부릅니다. 이 함수가 바로 서버측 개발자가 구현해야 할 함수입니다. 이 process 함수 안에서 제공된 nameNvalue_ 변수를 이용하여, requested_document_에 따라 이미 템플릿 값으로 생성되어 있는 content_를 변형시키는 것이 목적입니다. 만약 서버측에서 아무런 해야 할 일이 없다면 아무 것도 하지 않아도 됩니다. 튜토리얼 1에서 이러한 정적인 웹페이지의 예를 볼 수 있습니다.
